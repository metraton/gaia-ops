# Gaia-Ops Templates

**[Version en espanol](README.md)**

Templates used during installation to generate configuration files for each project.

## Purpose

Provide a consistent base for configuration files. `gaia-init` and `gaia-update` use these templates to generate or update project files.

## How It Works

```
[gaia-init] collects project data
        |
Reads template/
        |
Static copy (CLAUDE.md), merge (settings.json), or interpolation (governance.template.md)
        |
Generates final file in project
```

## Available Templates

### `CLAUDE.template.md` (~66 lines)

Static template for the orchestrator file. Copied as-is by `gaia-init` and `gaia-update` — no placeholder interpolation.

Contains: identity, routing table, agent descriptions, task tool syntax, work unit rule, processing agent responses.

**Generated by:** `bin/gaia-init.js`
**Output file:** `./CLAUDE.md`

---

### `settings.template.json` (~727 lines)

Template for permissions and security configuration. On update, new rules are merged while preserving user additions.

**Generated by:** `bin/gaia-init.js`
**Output file:** `./.claude/settings.json`

---

### `governance.template.md`

Template for the project governance document. Placeholders are filled with values from `project-context.json` on first use.

**Contains:**
- Stack Definition with placeholders: `[CLOUD_PROVIDER]`, `[PRIMARY_REGION]`, `[PROJECT_ID]`, `[CLUSTER_NAME]`, `[GITOPS_PATH]`, `[TERRAFORM_PATH]`, `[POSTGRES_INSTANCE]`, `[CONTAINER_REGISTRY]`
- 6 immutable architectural principles (Code-First, GitOps SSOT, Security Tiers, Conventional Commits, Workload Identity, Resource Limits)
- ADR process and standards evolution sections

**Generated by:** `speckit.init` skill (triggered automatically by `speckit-planner` on first speckit use)
**Synced by:** `speckit-planner` agent at the start of every session (automatic Step 0)
**Output file:** `<speckit-root>/governance.md`

## Usage

### During Installation

```bash
npx gaia-init
# Reads templates/, generates files
```

### Regeneration

```bash
# Regenerate via postinstall
node node_modules/@jaguilar87/gaia-ops/bin/gaia-update.js
```
