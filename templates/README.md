# Gaia-Ops Templates

Templates used during installation to generate configuration files for each project.

## How It Works

```
[gaia-scan] collects project data
        |
Reads templates/
        |
Static copy (settings.json), merge, or interpolation (governance.template.md)
        |
Generates final file in project
```

## Available Templates

### `settings.template.json`

Template for hooks configuration only. Contains 9 hook types (PreToolUse, PostToolUse, SubagentStop, SessionStart, UserPromptSubmit, Stop, TaskCompleted, SubagentStart, PostCompact) and environment settings. **No permissions** — deny rules and allow lists are merged into `settings.local.json` by `gaia-update.js` (NPM) or `session_start.py` (plugin).

**Generated by:** `bin/gaia-scan.py`, `bin/gaia-update.js`
**Output file:** `./.claude/settings.json`

---

### `managed-settings.template.json`

Reference template for enterprise deployment via Claude.ai Admin Console or `/etc/claude-code/managed-settings.json`. Contains wildcard deny rules with highest precedence (cannot be overridden by user/project settings). Includes `disableBypassPermissionsMode` to prevent `--dangerously-skip-permissions`.

**Deployed by:** Organization admin (not automated)
**Output:** `/etc/claude-code/managed-settings.json` (Linux) or Admin Console

---

### `governance.template.md`

Template for the project governance document. Placeholders are filled with values from `project-context.json` on first use.

**Contains:**
- Stack Definition with placeholders: `[CLOUD_PROVIDER]`, `[PRIMARY_REGION]`, `[PROJECT_ID]`, `[CLUSTER_NAME]`, `[GITOPS_PATH]`, `[TERRAFORM_PATH]`, `[POSTGRES_INSTANCE]`, `[CONTAINER_REGISTRY]`
- 6 immutable architectural principles (Code-First, GitOps SSOT, Security Tiers, Conventional Commits, Workload Identity, Resource Limits)
- ADR process and standards evolution sections

**Generated by:** `gaia-scan` (step 4.8, automatic on install). Updated by: `speckit.init` (Step 0, always on execution).
**Logic:** If not exists, create full file. If exists, update only `## Stack Definition`, preserve principles and ADRs.
**Output file:** `<speckit-root>/governance.md` (speckit-root defined in `project-context.json` under `paths.speckit_root`)

## Note on CLAUDE.md

Orchestrator identity is no longer generated from a template. It is injected dynamically by the UserPromptSubmit hook via `ops_identity.py` and on-demand skills (`project-dispatch`, `agent-response`).

## Usage

```bash
# During installation
npx gaia-scan

# Regeneration via postinstall
node node_modules/@jaguilar87/gaia-ops/bin/gaia-update.js
```
