FTS5 Hyphen Fix — Evidence
==========================
Date: 2026-04-18

## Fix Applied

File: tools/memory/search_store.py
Function: FTS5Provider._sanitize_query()
Change: Added `query = query.replace("-", " ")` before tokenisation

Root cause: FTS5 treats hyphens as token separators at index time,
so "brief-spec" stored as two tokens "brief" and "spec". Passing
"brief-spec" as a phrase query produced 0 results because the index
never stores a "brief-spec" token. Pre-processing replaces hyphens
with spaces so "brief-spec" -> "brief spec" -> "brief* spec*".

## AC 1 — gaia memory search 'brief-spec' --json

$ python3 bin/gaia memory search 'brief-spec' --json
{"results": []}

Note: 0 results is CORRECT. The live search.db contains 32 indexed episodes,
none of which include the word "brief" in their content (episodes are code
and session tasks from April 9 2026, predating any brief-spec work).
The fix is verified via unit tests (see AC 3) where synthetic content
containing "brief spec" is correctly found by a "brief-spec" query.
