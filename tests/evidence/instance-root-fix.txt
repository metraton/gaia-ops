## Implementation Notes

### Root cause of the original bug

`_find_project_root()` and `_resolve_db_path()` both stopped at the FIRST
`.claude/` found walking upward from cwd. Running from `gaia-ops-dev/` found
`gaia-ops-dev/.claude/` first, yielding the dev database (20 episodes).

### Why the original "walk to HOME" approach needed refinement

`$HOME/.claude/` exists as Claude Code's user settings directory.
Walking all the way to HOME and picking the highest `.claude/` would return
`/home/jorge` — but that dir's `.claude/` has no episodic memory data (no
`index.json`). It is a CC settings dir, not a Gaia instance.

### Final approach: Gaia marker discrimination

`find_highest_claude_root()` (in tools/memory/paths.py) now:
1. Walks from cwd up to Path.home() (inclusive)
2. Collects candidates that have a `.claude/` AND at least one Gaia marker
   (`hooks/`, `agents/`, or `skills/`) under it
3. Returns the highest (closest to HOME) Gaia-qualified candidate
4. Falls back to highest plain `.claude/` if no Gaia-qualified one found

This discriminates Gaia instance roots from the CC user settings dir at $HOME.
